# Academic Topic Research Agent

> 把一句话研究想法，变成导师能继续聊、数据能落地、投稿有路线的中文开题包。

很多 AI 调研工具会给你一堆论文链接和一段综述。
但真正卡住研究生的通常不是"有没有论文"，而是：

- 这个方向到底有没有人做过？
- 我的选题和前人工作差在哪？
- 数据集或研究设计撑不撑得住？
- 导师会不会说"这只是 XX 概念换皮"？
- 这个题适合 CHI / CSCW / LAK / AIED，还是其实应该换 venue？
- 标题、摘要、RQ、贡献写出来，会不会一眼像工程项目而不是研究问题？

**Academic Topic Research Agent** 是一个中文默认的科研选题前置调研 skill。
你给它一句话研究想法，它会按 quick / standard / deep 三档深度，生成一份可以拿去和导师讨论的选题尽调包。

## 它会产出什么？

一轮完整运行后，你会得到：

- 一份中文 v2 风格选题报告：Markdown + LaTeX，本机有 TeX 时附 PDF
- 文献元数据矩阵：题目、作者、年份、venue、方法、数据集、贡献、limitations、和你课题的关系
- 数据 / 研究可行性判断：Pass / Weak Pass / Fail
- 3–8 个标题候选
- 1–3 段摘要草稿
- 研究问题、方法边界、评价路线、风险表和下一步计划
- 面向目标 venue 的投稿定位和 reviewer attack 预判

## 它不是普通"帮我综述一下"

普通模式帮你回答：
**这个方向有什么、差在哪、能不能做。**

Researcher-grade 模式会继续追问：
**这个选题能不能经得起导师和审稿人攻击。**

它会运行 Module H：构念压力测试 + 8 角 critique + 重写循环 + venue fit 决策，专门检查这些高频翻车点：

1. 你的论文是不是有多个 headline 在抢戏？
2. 你的核心概念是不是已有概念换皮？
3. 你的 level / ladder 是不是偷偷压扁了多个维度？
4. formative study 是真实 inquiry，还是提前写好答案的 performance？
5. ground truth 是否稳定？专家不同意时怎么办？
6. 部署成本是不是被你包装成了"低打扰"？
7. 系统名、条件名、访谈问题有没有预埋答案？
8. CHI / CSCW / LAK / AIED 到底谁会最喜欢这个贡献？

## 三档深度

| 深度 | 适合场景 | 大致规模 |
|---|---|---|
| `quick` | 临时想法体检，半小时知道坑在哪 | 10–20 篇 |
| `standard` | 导师下周要看的前期调研，默认推荐 | 30–50 篇 |
| `deep` | 开题、博士选题、完整文献网、researcher-grade 打磨 | 80–120 篇 |

`deep` 会默认开启 researcher-grade 模式；`standard` 可以用关键词触发：
`researcher-grade` / `深度打磨` / `构念压力测试` / `CHI 级` / `CSCW 转身` / `期刊投稿级`。

## 适合谁？

- 研一 / 博一正在找方向
- 开题前需要给导师一份像样的前期调研
- HCI / CSCW / LAK / L@S / AIED / EDM / IJHCI 等方向选题
- 想确认一个 idea 是研究问题，还是只是工程 demo
- 已经有模糊想法，但说不清 contribution、dataset、venue fit

## 不适合谁？

- 想让 AI 直接替你写论文
- 想编造引用、实验结果或不存在的数据集
- 想下载付费 PDF
- 想让工具替你做最终学术判断

它会帮你把"该做的脏活"前置做好，但不会替你决定这个方向值不值得赌。
